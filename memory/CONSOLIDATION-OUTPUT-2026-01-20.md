## Pattern 12 Reference

### Quick Reference: Event Delegation for Multi-Item Features

**When to Use**:
- Multiple similar items (courses, tools, discussions, buttons)
- Need for scalable architecture (works with 5, 50, or 5000 items)
- Dynamic content rendering (listeners don't need per-item setup)

**Implementation**:
```javascript
document.addEventListener('click', (event) => {
  const element = event.target.closest('[data-action]');
  if (!element) return;
  const action = element.dataset.action;
  const id = element.closest('[data-id]')?.dataset.id;
  routeHandler(action, id); // Routes to appropriate function
});
```

**Performance**:
- O(1) complexity regardless of item count
- Single listener replaces N listeners (where N = number of items)
- Works on dynamically added elements (no re-registration needed)

**Data Attributes**:
- `data-action="open"` → Route action to handler
- `data-coming-soon="true"` → Feature flag
- `data-filter="in-progress"` → Filter value
- `data-view="grid|list"` → Layout mode
- `data-id="123"` → Item identifier

**Anti-Pattern**:
- Individual event listeners per item (doesn't scale)
- Large if/else chains in click handler (use switch + data attributes)
- Mixing behavior in CSS classes (use data attributes for logic)
